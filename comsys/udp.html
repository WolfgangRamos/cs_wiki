--- layout: page title: User Datagram Protocol ---

erbrachter Dienst: Pakete unzuverlässig zwischen Anwendungen auf Endsystemen übertragen

  • verbindungsloses Protokoll
  • Verwendet für Anwendungen, bei denen das Korrigieren von Übertragungsfehlern schlechter ist, als das Vorhandensein von Fehlern. Bsp: VOIP, Multimedia streaming

Vorteile gegenüber TCP

  • keine Verzögerung durch Aufbau einer Verbindung
  • kleiner Header
  • keine congestion control (segments können so schnell wie möglich gesendet werden)
